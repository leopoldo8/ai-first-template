# Spec-Driven Development

## TLC Methodology

The TLC (Tech Leads Club) Spec-Driven Development methodology structures work into adaptive phases:

1. **Specify** — Define WHAT and WHY (always required)
2. **Design** — Define HOW (Large/Complex only)
3. **Tasks** — Break into atomic steps (Large/Complex only)
4. **Execute** — Implement with TDD (always required)

## Integration with Brainstorming

For unclear features, use `superpowers:brainstorming` first to explore approaches. The brainstorming output (in `docs/superpowers/specs/`) feeds into the TLC Specify phase.

## Persistent Memory

`.specs/project/STATE.md` preserves decisions, blockers, and context across sessions.
